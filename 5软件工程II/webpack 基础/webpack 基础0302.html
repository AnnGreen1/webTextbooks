                                                                                                                      一、入门

                                                                                                                      1、初识 webpack

                                                                                                                      webpack ，是一个流行的前端项目构建工具，是JavaScript 应用程序的静态模块打包工具。它能够根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规则生成对应的静态资源。

                                                                                                                      当 webpack 处理应用程序时，它会在内部构建一个 依赖图 (dependency graph)，此依赖图会映射项目所需的每个模块，并生成一个或多个 bundle。webpack 可以将多种静态资源 js、css、less 转换成一个静态文件，减少了页面的请求。

                                                                                                                      为什么要使用 webpack ？

                                                                                                                      在打包工具出现之前，我们是如何在 web 中使用 JavaScript 的。

                                                                                                                      在浏览器中运行 JavaScript 一般有两种方法：

                                                                                                                      1. 引用一些脚本来存放每个功能；不过这种方式有一定的缺陷，一旦加载的脚本过多，那么就会导致网络瓶颈问题。
                                                                                                                      2. 第二种方式，使用一个包含所有项目代码的大型 .js 文件，但是这会导致作用域、文件大小、可读性和可维护性等方面的问题。

                                                                                                                      webpack，可以打包你的 JavaScript 应用程序（支持 ESM 和 CommonJS），可以扩展为支持许多不同的资源，例如：images, fonts 和 stylesheets。

                                                                                                                      Webpack打包工具对项目中的复杂文件进行打包处理，可以实现项目的自动化构建，并且给前端开发人员带来了极大的便利。目前，企业中的绝大多数前端项目是基于Webpack打包工具来进行开发的。

                                                                                                                      2、webpack 核心概念

                                                                                                                      2.1、入口（entry）

                                                                                                                      入口起点（entry point）用来指示 webpack 应该使用哪个模块来作为 构建其内部依赖图(dependency graph) 的开始。进入入口起点后，webpack 会找出有哪些模块和库是入口起点（直接和间接）依赖的。

                                                                                                                      2.1.1、语法

                                                                                                                      1、单个入口的语法格式：

                                                                                                                      entry: string|Array<string>

                                                                                                                      默认值是： ./src/index.js，我们也可以通过在 webpack configuration 中配置 entry 属性，来指定一个（或多个）不同的入口起点。例如：

                                                                                                                      webpack.config.js

                                                                                                                      xxxxxxxxxx
                                                                                                                      // 简写形式
                                                                                                                      module.exports = {
                                                                                                                        // 配置入口
                                                                                                                        entry: './path/to/my/entry/file.js'
                                                                                                                      };

                                                                                                                      Tips 我们也可以给 entry 传入一个数组。向 entry 属性传入文件路径数组后，将创建出一个 多主入口(multi-main entry)。如果我们想要一次注入多个依赖文件，并且将它们的依赖导向(graph)到一个 chunk 时，就可以用这种方式。

                                                                                                                      2、对象语法

                                                                                                                      xxxxxxxxxx
                                                                                                                      entry: {[entryChunkName: string]: string|Array<string>}

                                                                                                                      这是应用程序中定义入口的最可扩展的方式。即可以让这些配置可以重复使用，并且可以与其他配置组合使用。

                                                                                                                      webpack.config.js

                                                                                                                      xxxxxxxxxx
                                                                                                                      module.exports = {
                                                                                                                        // 配置入口
                                                                                                                        entry: {
                                                                                                                          app: './src/app.js',
                                                                                                                          adminApp: './src/adminApp.js'
                                                                                                                        }
                                                                                                                      };

                                                                                                                      2.2、出口（output）

                                                                                                                      output 属性告诉 webpack 在哪里输出它所创建的 bundle，以及如何命名这些文件（如bundle、asset 和其他你所打包或使用 webpack 载入的任何内容等），可以控制 webpack 如何向硬盘写入编译文件。

                                                                                                                      主要输出文件的默认值是 ./dist/main.js，其他生成文件默认放置在 ./dist 文件夹中。

                                                                                                                      注意 可以存在多个入口起点，但只指定一个输出配置。

                                                                                                                      2.2.1、语法

                                                                                                                      在 webpack 中配置 output 属性的最低要求是，将它的值设置为一个对象。 我们在配置中去指定一个 output 字段，来配置这些处理过程：

                                                                                                                      webpack.config.js

                                                                                                                      xxxxxxxxxx
                                                                                                                      const path = require('path'); //  Node.js 核心模块，主要用来操作文件路径。
                                                                                                                      

                                                                                                                      module.exports = {
                                                                                                                        // 配置入口
                                                                                                                        entry: './path/to/my/entry/file.js',
                                                                                                                        // 配置出口
                                                                                                                        output: {
                                                                                                                          path: path.resolve(__dirname, 'dist'), // 对应一个绝对路径
                                                                                                                          filename: 'bundle.js'
                                                                                                                        }
                                                                                                                      };

                                                                                                                      编译文件的文件名(filename)，首选推荐： main.js || bundle.js || index.js

                                                                                                                      output.path 对应一个绝对路径，此路径是你希望一次性打包的目录。

                                                                                                                      2.3、loader

                                                                                                                      loader 用来 对模块的源代码进行转换。因为 webpack 只能识别 JavaScript 和 JSON 文件，所以就需要通过 loader 将不能识别的部分转换为有效模块，让 webpack 能够去处理其他类型的文件，以供应用程序使用，以及被添加到依赖图中。

                                                                                                                      loader 可以将文件从不同的语言（如 TypeScript）转换为 JavaScript 或将内联图像转换为 data URL。loader 甚至还可以允许我们直接在 JavaScript 模块中 import CSS文件。

                                                                                                                      我们可以通过以下两个属性来配置，这两个属性也是必须要设置的：

                                                                                                                      • test 属性：用于标识出应该被对应的 loader 进行转换的某个或某些文件。
                                                                                                                      • use 属性：在进行转换时，指定应该使用哪个 loader。
                                                                                                                      2.3.1、使用方式

                                                                                                                      在应用程序中有三种使用 loader 的方式：

                                                                                                                      • 配置（推荐）：在 webpack.config.js 文件中指定 loader。
                                                                                                                      • 内联：在每个 import 语句中显式指定 loader。
                                                                                                                      • CLI：在 shell 命令中指定 loader。
                                                                                                                      2.3.1.1、在 webpack.config.js 配置（推荐）

                                                                                                                      我们在 module.rules 中对loader进行配置，允许我们在 webpack 配置中指定多个 loader。 这种方式是展示 loader 的一种简明方式，并且有助于使代码变得简洁和易于维护：

                                                                                                                      xxxxxxxxxx
                                                                                                                      const path = require('path');
                                                                                                                      

                                                                                                                      module.exports = {
                                                                                                                        output: {
                                                                                                                          filename: 'bundle.js'
                                                                                                                        },
                                                                                                                        module: {
                                                                                                                          rules: [
                                                                                                                            { test: /\.txt$/, use: 'raw-loader' } 
                                                                                                                            // 此处test使用了正则表达式，表示匹配任何以 .txt 结尾的文件
                                                                                                                            // 此处use指定了用raw-loader来转换
                                                                                                                          ]
                                                                                                                        }
                                                                                                                      };

                                                                                                                      在以上配置中，我们对一个 module 对象定义了 rules 属性，里面包含两个必须属性：test 和 use。这段配置会让 webpack 编译器(compiler) 在遇到 .txt 的路径时，就会在打包之前，使用 raw-loader 进行转换然后打包。

                                                                                                                      注意

                                                                                                                      loader 是从右到左地取值(evaluate)/执行(execute)。

                                                                                                                      2.3.1.2、内联方式进行配置

                                                                                                                      在 import 语句或任何等同于 "import" 的方法中指定 loader。使用 ! 将资源中的 loader 分开，每个部分都会相对于当前目录解析。

                                                                                                                      xxxxxxxxxx
                                                                                                                      import Styles from 'style-loader!css-loader?modules!./styles.css';
                                                                                                                      2.3.1.3、通过 CLI 使用 loader：

                                                                                                                      我们还可以通过 CLI（命令行服务）来使用 loader。

                                                                                                                      如下命令：

                                                                                                                      xxxxxxxxxx
                                                                                                                      webpack --module-bind jade-loader --module-bind 'css=style-loader!css-loader'

                                                                                                                      这段命令会对 .jade 文件使用 jade-loader； 会对 .css 文件使用 style-loader 和 css-loader。

                                                                                                                      2.4、插件（plugin）

                                                                                                                      插件（plugin）是 webpack 的支柱功能。主要是用来 解决 loader 无法实现的其他事。比如：打包优化，资源管理，注入环境变量等等。总之通过插件接口(plugin interface) ，可以帮助我们处理各种各样的任务。

                                                                                                                      2.4.1、使用方式
                                                                                                                      通过 require() 引用插件

                                                                                                                      在应用程序中如果想要使用一个插件，我们只需要两步：

                                                                                                                      1. require() 它；
                                                                                                                      2. 把它添加到 plugins 数组中；

                                                                                                                      我们可以在一个配置文件中多次使用同一个插件，通过使用 new 操作符来创建插件的实例 即可。 由于插件可以携带参数/选项，所以我们必须在 webpack 配置中向 plugins 属性传入 new 出来的实例。

                                                                                                                       

                                                                                                                      xxxxxxxxxx
                                                                                                                      const HtmlWebpackPlugin = require('html-webpack-plugin'); // 此插件需要先通过 npm 安装
                                                                                                                      const webpack = require('webpack'); // 用于访问内置插件
                                                                                                                      

                                                                                                                      module.exports = {
                                                                                                                        module: {
                                                                                                                          rules: [
                                                                                                                            { test: /\.txt$/, use: 'raw-loader' }
                                                                                                                          ]
                                                                                                                        },
                                                                                                                        plugins: [
                                                                                                                          new HtmlWebpackPlugin({template: './src/index.html'})
                                                                                                                        ]
                                                                                                                      };

                                                                                                                      在以上代码中，我们使用的内置插件 html-webpack-plugin ，主要用来为应用程序生成 HTML 一个文件，并自动注入到所有生成的 bundle中。

                                                                                                                      2.5、模式（mode）

                                                                                                                      webpack 给我们提供 mode 配置选项，能够让 webpack 使用相应环境的内置优化。一共有3种模式：

                                                                                                                      1. development：开发模式
                                                                                                                      2. production：（默认）生产模式
                                                                                                                      3. none ：无

                                                                                                                      2.5.1、用法

                                                                                                                      我们只要在配置对象中提供 mode 选项即可：

                                                                                                                      xxxxxxxxxx
                                                                                                                      module.exports = {
                                                                                                                        mode: 'development'
                                                                                                                      };

                                                                                                                      也可以通过CLI设置：

                                                                                                                      xxxxxxxxxx
                                                                                                                      webpack --mode=development

                                                                                                                      2.6、浏览器兼容性

                                                                                                                      webpack 支持所有符合 ES5 标准的浏览器（不支持 IE8 及以下版本）。webpack 的 import() 和 require.ensure() 需要 Promise。如果我们想要支持旧版本浏览器，那么在使用这些表达式之前，还需要需要提前加载 polyfill。

                                                                                                                      3.1、本地化安装

                                                                                                                      3.1.1、环境 Node.js

                                                                                                                      首先终端上输入：node -v 、 npm -v 可查看当前安装版本

                                                                                                                      如果没有安装，则通过中文本地址下载安装。

                                                                                                                      3.1.2、创建项目目录

                                                                                                                      通过终端，首先进入需要创建项目的目录中，我们在桌面创建项目所以直接进入桌面然后再创建项目目录，名称是：webpack-demo，并进入该目录：

                                                                                                                      xxxxxxxxxx
                                                                                                                      mkdir webpack-demo && cd webpack-demo

                                                                                                                      注意 建议大家不要直接打开终端后就开始创建，建议大家先进入你们需要存放项目的位置，比如我是在桌面创建，所以先通过cd desktop进入桌面，然后再创建目录。

                                                                                                                      3.1.3、创建 package.json 文件

                                                                                                                      package.json 文件是一个标准的npm说明文件，蕴含了丰富的信息，包括当前项目的依赖模块，自定义的脚本任务等等。

                                                                                                                      接下去我们需要生成 package.json 文件， 通过以下命令即可创建：

                                                                                                                      xxxxxxxxxx
                                                                                                                      npm init -y

                                                                                                                      Tips：

                                                                                                                      命令中的 -y ，相当于 --yes，全部使用默认的值

                                                                                                                      接着就会在我们的目录中创建好 package.json 文件。

                                                                                                                      3.1.4、安装 webpack

                                                                                                                      在终端输入命令进行安装：

                                                                                                                      xxxxxxxxxx
                                                                                                                      npm i -D webpack

                                                                                                                      注意 对于 webpack 4.0.0之前的版本，我们只要安装 webpack 之后可以直接使用了。 但是，从4.0.0开始，webpack 有一部分被封装到了 webpack-cli 中了，而目前我们通过此方式安装的是4之后的版本，因此需要再额外安装 webpack-cli 。

                                                                                                                      3.1.5、安装 webpack-cli

                                                                                                                      通过以下命令来安装 webpack-cli：

                                                                                                                      xxxxxxxxxx
                                                                                                                      npm i -D webpack-cli

                                                                                                                      到此为止，已经完成了项目的准备工作了。

                                                                                                                      Tips：

                                                                                                                      npm中的参数：

                                                                                                                      -D、--save-dev：把模块版本信息保存到devDependencies（开发环境依赖）中，即你的package.json文件的devDependencies字段中：

                                                                                                                      -S、--save：把模块的版本信息保存到dependencies（生产环境依赖）中，即你的package.json文件的dependencies字段中。

                                                                                                                      更多命令及简写，可参考 https://xovel.cn/article/npm-alias.html

                                                                                                                      4、webpack 初体验

                                                                                                                      4.1、创建相应项目结构

                                                                                                                      在项目开始前，我们先在项目目录中搭建相应的目录结构，分别新建 dist 和 src 目录：

                                                                                                                      • src文件夹：用来存放原始数据和我们将写的JavaScript模块，并添加 index.js 文件，后续我们需要以项目根目录下的./src/index.js作为入口的。
                                                                                                                      • dist文件夹：用来存放最终打包好的供浏览器读取的文件，并添加 index.html 文件。

                                                                                                                      4.2、安装lodash依赖和添加js文件

                                                                                                                      lodash是生产环境下依赖，是一个具备一致性、模块化、高性能的 JavaScript 实用工具库。通过以下命令安装：

                                                                                                                      xxxxxxxxxx
                                                                                                                      npm i lodash --save

                                                                                                                      4.3、为 index.js 和 index.html 添加内容

                                                                                                                      我们可以在 ./src/index.js 中添加一些测试内容：

                                                                                                                        let element = document.createElement('div');
                                                                                                                        element.innerHTML = _.join(['Hello', 'webpack'], ' '); // _.join()主要用来将数组转化成字符串，每个元素之间使用第二个参数连接
                                                                                                                        return element;
                                                                                                                      }
                                                                                                                      

                                                                                                                      document.body.append(createDomElement());

                                                                                                                      在 ./dist/index.html 中添加 h5 模版，在vscode中，可以通过输入 !，后再按 tab 键去生成。

                                                                                                                      </body>
                                                                                                                      </html>

                                                                                                                      4.4、添加 webpack 配置文件

                                                                                                                      在根目录中新建 webpack.config.js 文件，该文件为webpack默认的配置文件。

                                                                                                                      xxxxxxxxxx
                                                                                                                      const path = require('path'); // 存放的相对路径
                                                                                                                      

                                                                                                                      module.exports = { // 对外导出对象
                                                                                                                        entry: './src/index.js',
                                                                                                                        mode: 'development',
                                                                                                                        output: {
                                                                                                                          filename: 'main.js', // 最终打包生成后的文件名

                                                                                                                        }
                                                                                                                      }

                                                                                                                      4.5、运行

                                                                                                                      通过命令npx可以执行node_modules中 bin 目录中的二进制文件。

                                                                                                                      xxxxxxxxxx
                                                                                                                      npx webpack

                                                                                                                      成功后就会在 dist 目录中生成 main.js，其中的代码是打包好的代码

                                                                                                                      模块(module)

                                                                                                                      在模块化编程中，将程序分解为功能离散的 “chunk” (discrete chunks of functionality)，称之为 模块。在webpack中通过 loader 可以支持通过各种模块。

                                                                                                                      loader 实际上是描述了 webpack 如何去处理非 JavaScript 模块，并且如何在 bundle 中引入这些依赖。

                                                                                                                      webpack 模块可以支持的有以下几种：

                                                                                                                      • ES2015 import 语句，即 ES6 的 import（建议使用）
                                                                                                                      • CommonJS require() 语句
                                                                                                                      • AMD define 和 require 语句
                                                                                                                      • css/sass/less 文件中的 @import 语句
                                                                                                                      • 样式 url(...) 或 HTML 文件 <img src=...> 中的图片链接（image url）
                                                                                                                      module.rules

                                                                                                                      创建模块时，用来匹配请求规则的数组。这些规则可以修改模块的创建方式，可以对模块（module）应用 loader，以及修改解析器（parse）

                                                                                                                      xxxxxxxxxx
                                                                                                                      module.exports = { // 对外导出对象
                                                                                                                       
                                                                                                                          module: {
                                                                                                                              rules: [ // 规则，各种模块的规则都放在其中，因此对应的是一个数组，
                                                                                                                                  {
                                                                                                                                      test: /\.css$/, // 正则表达式，用来匹配后缀名为.css的文件 
                                                                                                                                      use: ['style-loader', 'css-loader'] // 指定使用的loader。表示上面的test模块，用这里use指定的loader进行处理
                                                                                                                                  }
                                                                                                                              ]
                                                                                                                          }
                                                                                                                      }

                                                                                                                      Rule 条件详解：

                                                                                                                      • 字符串：匹配输入必须以提供的字符串开始，目录绝对路径或文件绝对路径。
                                                                                                                      • 正则表达式：test 输入值
                                                                                                                      • 函数：调用输入的函数。必须返回一个真值（truthy value）以匹配
                                                                                                                      • 条件数组：至少匹配一个条件
                                                                                                                      • 对象：匹配所有属性，每个属性都有一个定义行为

                                                                                                                      Rule.test

                                                                                                                      {test: Condiction}： 匹配特定条件。一般是提供一个正则表达式或正则表达式的数组，不过这也不是非必需的。

                                                                                                                      我们还可以使用以下的条件：

                                                                                                                      • {include: Condition}：匹配特定条件。一般是提供一个字符串或者字符串数组，但这不是强制的
                                                                                                                      • {exclude: Condition}：排除特定条件。一般是提供一个字符串或字符串数组，但这不是强制的
                                                                                                                      • {and: Condition}：必须匹配数组中的所有条件
                                                                                                                      • {or: COndition}：匹配数组中任何一个条件
                                                                                                                      • {not: Condition}：必须排除这个条件
                                                                                                                      xxxxxxxxxx
                                                                                                                      module.exports = { // 对外导出对象
                                                                                                                       
                                                                                                                          module: {
                                                                                                                              rules: [ // 规则，各种模块的规则都放在其中，因此对应的是一个数组，
                                                                                                                                  {
                                                                                                                                      test: /\.css$/, // 正则表达式，用来匹配后缀名为.css的文件
                                                                                                                                      include: [
                                                                                                                                          path.resolve(__dirname, 'app/styles'),
                                                                                                                                          path.resolve(__dirname, 'vendor/styles')
                                                                                                                                      ] // 包含这些路径时，使用test进行匹配
                                                                                                                                      use: ['style-loader', 'css-loader'] // 指定使用的loader。表示上面的test模块，用这里use指定的loader进行处理
                                                                                                                                  }
                                                                                                                              ]
                                                                                                                          }
                                                                                                                      }

                                                                                                                      Rule.use

                                                                                                                      主要应用于模块指定使用的loader。

                                                                                                                      支持链式传递，不过需要注意的是： 从右向左、从下到上 应用到模块上的：

                                                                                                                      xxxxxxxxxx
                                                                                                                      module.exports = { // 对外导出对象
                                                                                                                       
                                                                                                                          module: {
                                                                                                                              rules: [ // 规则，各种模块的规则都放在其中，因此对应的是一个数组，

                                                                                                                                          {
                                                                                                                                              loader: 'css-loader'

                                                                                                                      什么是Source map？

                                                                                                                      通过webpack，我们可以实现对代码进行打包压缩，这样大大减少了代码大小，从而有效提高访问速度。

                                                                                                                      不过，如果压缩后的代码一旦报错，那么仅仅根据报错信息是很难进行Debug的，主要是由于被压缩后它的行号和列号已经失真了，所以，我们就需要通过 Source Map，来还原代码真实的出错位置了。

                                                                                                                      如上案例，当我们选中 .box 这个元素时，在开发者工具上并不会显示该元素的样式在哪个文件哪一行定义的。

                                                                                                                        module: {
                                                                                                                          rules: [ // 规则，各种模块的规则都放在其中，因此对应的是一个数组

                                                                                                                      7、打包其他资源

                                                                                                                      其他资源: 不需要做任何处理，原封不动的打包出去(以字体图标为例iconfont)

                                                                                                                      在之前的 webpack4 中，打包其他资源需要使用 file-loader， 不过在webpack5打包其他资源已经可以不用到 file-loader与 url-loader 了。 webpack5可以使用资源模块类型（asset module type）来替代 file-loader 等，以上的字体的打包就是一个例子。

                                                                                                                      7.1、资源模块类型

                                                                                                                      资源模块类型(asset module type)，通过添加 4 种新的模块类型，来替换所有这些 loader：

                                                                                                                      • asset/resource 发送一个单独的文件并导出 URL。之前通过使用 file-loader 实现。
                                                                                                                      • asset/inline 导出一个资源的 data URI。之前通过使用 url-loader 实现。
                                                                                                                      • asset/source 导出资源的源代码。之前通过使用 raw-loader 实现。
                                                                                                                      • asset 在导出一个 data URI 和发送一个单独的文件之间自动选择。之前通过使用 url-loader，并且配置资源体积限制实现。

                                                                                                                      8、Web 服务

                                                                                                                      大家是否有注意到，在开发中我们每次编写好代码之后，都需要执行npm run build（npx webpack）命令来更新js等文件，重新打包后才会刷新页面，才能看到更新效果，这种开发调试的方式效率并不高，webpack-dev-server 正好解决了此问题，是一款便捷的本地开发工具。

                                                                                                                      特点：

                                                                                                                      • 配置完后可以自动编译、自动打开浏览器、自动刷新浏览器
                                                                                                                      • 只会在内存中编译打包，不会有任何输出

                                                                                                                      webpack-dev-server 是webpack官方提供的一个小型 Express 服务器，使用它可以为webpack打包生成的资源文件提供web服务：

                                                                                                                      1. 为静态文件提供web服务

                                                                                                                      2. 自动刷新和热替换(HMR)

                                                                                                                        • 自动刷新指当修改代码时 webpack 会进行自动编译，更新网页内容
                                                                                                                        • 热替换指运行时更新各种模块，即局部刷新。

                                                                                                                      当我们对代码进行修改并保存后，webpack 将对代码重新打包，并将新的模块发送到浏览器端，浏览器通过新的模块替换老的模块，这样在不刷新浏览器的前提下就能够对应用进行更新。

                                                                                                                      三、webpack 性能优化

                                                                                                                      1、oneOf

                                                                                                                      我们在使用 webpack 时，对于非 JavaScript 代码的模块无法直接使用，我们是通过 loader 来帮助我们解析处理的。

                                                                                                                      一般来说，对于同类型的文件通常是只交给一个 loader 来处理，那么，按照正常逻辑来看的话，一个 css 文件，当匹配到 test 为 .css/.sass/.scss 后缀的loader 后就应该要立即解析了。然而在 webpack 中，默认是即使已经匹配到了，仍然还是会完整遍历一遍后再进行解析，每个文件都会一个一个地走一遍，这样无疑会拖慢编译构建速度。

                                                                                                                      而 Oneof 语法就是解决这个问题的，使文件一旦匹配上 loader 之后就立即解析，省去了全盘遍历这个不必要的过程。

                                                                                                                      1.1、语法格式

                                                                                                                      xxxxxxxxxx
                                                                                                                      module: {
                                                                                                                          rules: [
                                                                                                                              {//loader...},
                                                                                                                              {
                                                                                                                                  oneOf:[{//loader1...},{//loader2...}]
                                                                                                                              }
                                                                                                                          ]
                                                                                                                      }

                                                                                                                      1.2、使用

                                                                                                                      如下案例：

                                                                                                                      xxxxxxxxxx
