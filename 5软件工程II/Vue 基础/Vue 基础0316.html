                                                                                                                            一、Vue.js 入门

                                                                                                                            1、初识Vue

                                                                                                                            Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

                                                                                                                            兼容性

                                                                                                                            Vue 不支持 IE8 及以下版本，因为 Vue 使用了 IE8 无法模拟的 ECMAScript 5 特性。但它支持所有兼容 ECMAScript 5 的浏览器。

                                                                                                                            更新日志

                                                                                                                            最新版本：3.2.31（2022-02-12）

                                                                                                                            对比其他框架

                                                                                                                            与AngularJS (Angular 1)相比较

                                                                                                                            Vue 的一些语法和 AngularJS 的比较相似 (例如 v-if vs ng-if)。因为 AngularJS 是 Vue 早期开发的灵感来源。不过，AngularJS 中存在的许多问题，在 Vue 中已经得到解决。

                                                                                                                            复杂性

                                                                                                                            在 API 与设计两方面上 Vue.js 都比 AngularJS 简单得多，因此你可以快速地掌握它的全部特性并投入开发。

                                                                                                                            灵活性和模块化

                                                                                                                            Vue.js 是一个更加灵活开放的解决方案。它可以允许我们用希望的方式去组织应用程序，不需要在任何时候都必须遵循 AngularJS 制定的规则，这让 Vue 能适用于各种项目。

                                                                                                                            数据绑定

                                                                                                                            AngularJS 使用双向绑定（ng-bind 或 {{}}），Vue 在不同组件间强制使用单向数据流。这使应用中的数据流更加清晰易懂。

                                                                                                                            指令与组件

                                                                                                                            在 Vue 中指令和组件分得更清晰。指令只封装 DOM 操作，而组件代表一个自给自足的独立单元——有自己的视图和数据逻辑。在 AngularJS 中，每件事都由指令来做，而组件只是一种特殊的指令。

                                                                                                                            2、安装 Vue

                                                                                                                            Vue.js 设计的初衷就包括可以被渐进式地采用。这意味着它可以根据需求以多种方式集成到一个项目中。

                                                                                                                            将 Vue.js 添加到项目中主要有四种方式：

                                                                                                                            1. 在页面上以 CDN 包 的形式导入。
                                                                                                                            2. 下载 JavaScript 文件并自行托管。
                                                                                                                            3. 使用 npm 安装它。
                                                                                                                            4. 使用官方的 CLI 来构建一个项目，它为现代前端工作流程提供了功能齐备的构建设置 (例如，热重载、保存时的提示等等)。

                                                                                                                            如果我们直接通过 <script> 标签引入，Vue 会被注册为一个全局变量。

                                                                                                                            方式1：下载 Vue.js 文件

                                                                                                                            Vue官方提供了两个版本的版本，开发版本和生产版本，生产版本是被压缩后的版本，删除了相应的警告信息。

                                                                                                                            我们开发使用，目前先下载开发版本使用。下载后会得到一个 Vue.js 文件。然后将此文件放入我们的工程目录中，并通过 script 标签引入即可使用了。

                                                                                                                            方式2：CDN 引入

                                                                                                                            Vue 给我们提供了相应的CDN地址，我们也可以通过CDN的方式直接使用：

                                                                                                                            制作原型或学习
                                                                                                                            xxxxxxxxxx
                                                                                                                            <script src="https://unpkg.com/vue@next"></script>
                                                                                                                            生产环境

                                                                                                                            推荐链接到一个明确的版本号和构建文件，以避免新版本造成的不可预期的破坏：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <script src="https://cdn.jsdelivr.net/npm/vue@3.2.31/dist/vue.global.min.js"></script>

                                                                                                                            3.1、Vue 挂载点

                                                                                                                            Vue 在处理页面时，主要是通过查找挂载点，然后处理挂载点中的内容的。

                                                                                                                            如何设置挂载点？

                                                                                                                            在Vue实例中，我们 通过 mount() 函数来指定挂载点

                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount('#app')

                                                                                                                            如上代码，我们指定了 id 为 app 的 DOM 元素做为 Vue 的挂载点，Vue只会处理该元素内部的内容。那么，对对于不在 #app内部的相邻的 #box 元素是并不会处理的。

                                                                                                                            指令

                                                                                                                            指令 (Directives) 是带有 v- 前缀的特殊 attribute（属性）。指令 attribute 的值预期是单个 JavaScript 表达式 (v-for 和 v-on 是例外情况，稍后我们再讨论)。一般指令的职责是当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM。以上案例中的 img 标签则是 v-html 和 v-bind 指令。

                                                                                                                            指令参数

                                                                                                                            一些指令是可以接受一个参数的，在指令名称后面用冒号表示，还是以上的 v-bind 指令：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <img v-bind:title="title" src="https://v3.cn.vuejs.org/logo.png">

                                                                                                                            在这里的 title 是参数，告知 v-bind 指令将该元素的 title attribute 与表达式 title 的值绑定。

                                                                                                                            再来看看另一个例子， v-on 指令，它用于监听 DOM 事件：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                <button v-on:click="hide()">隐藏 / 显示图片</button><br>
                                                                                                                                <img v-bind:title="title" v-show="isHide" src="https://v3.cn.vuejs.org/logo.png">
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      title: "Vue的图标",
                                                                                                                                      isHide: true
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  methods: {
                                                                                                                                    hide() {
                                                                                                                                      this.isHide = !this.isHide
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                }).mount('#app')
                                                                                                                              </script>

                                                                                                                            以上案例中，我们使用了 v-on 指令，它用来监听 DOM 事件，这里的参数 click 则是被监听的单击事件的事件名。

                                                                                                                            动态参数

                                                                                                                            我们也可以设置动态参数，在指令参数中使用 JavaScript 表达式，用方括号括将表达括起来即可：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <img v-bind:[attributeName]="url" />

                                                                                                                            那么这里的 attributeName 会被作为一个 JavaScript 表达式进行动态求值，求得的值将会作为最终的参数来使用。

                                                                                                                            例如，如果组件实例中有一个 data property attributeName，其值为 "src"，那么这个绑定将等价于 v-bind:src，如下：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                <img v-bind:[attr]="url" />
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      attr: "src",
                                                                                                                                      url: "https://v3.cn.vuejs.org/logo.png"
                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount('#app')
                                                                                                                              </script>

                                                                                                                            同样地，你可以使用动态参数为一个动态的事件名绑定处理函数：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                <button v-on:[event]="hide()">隐藏 / 显示图片</button><br>
                                                                                                                                <img v-bind:title="title" v-show="isHide" src="https://v3.cn.vuejs.org/logo.png">
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      event: "mouseover", //  鼠标悬停时
                                                                                                                                      title: "Vue的图标",
                                                                                                                                      isHide: true
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  methods: {

                                                                                                                            4、Vue 与 MVVM

                                                                                                                            4.1、MVC 与 MVVM

                                                                                                                            1、MVC设计模式（Model-View-Controller）

                                                                                                                            模型-视图-控制器，是一种比较常用也比较经典的设计模式，通过数据模型，控制器逻辑，视图展示将应用程序进行逻辑划分。传统的MVC模式下，用户和View交互，然后View将用户的操作传递给Controller，Controller根据业务逻辑对Model进行更新。而View以观察者方式来判断Model的变化，再来更新View自己，MVC的所有通信都是单向的。

                                                                                                                            分工：model主要负责数据模型，view主要负责用户界面视图展示，controller是view和model间通信的桥梁，负责UI展现和数据交互的逻辑控制。

                                                                                                                            优点：耦合度低，可以使系统层次清晰，职责分明，不同功能分层，重用性高，利于模块化，也便于维护。

                                                                                                                            缺点：由于Controller 依赖于 view，联系过于紧密不易测试；视图对模型数据的低效率访问；不适合中小规模的应用程序。

                                                                                                                            2、MVVM设计模式（Model-View-ViewModel）

                                                                                                                            模型-视图-视图模型，它本质上就是MVC 的改进版。随着我们应用重量级的日益增长，在一些典型的 MVC 应用里， 有许多逻辑被放在 Controller 里，导致控制器变得愈发繁重，为了不让控制器日益增大，也为了能便于测试管理，便出现了MVVM。为了减轻Controller层和View层的压力实现更加清晰化代码，将相关的表示逻辑从 Controller 移出放到一个新的对象里，即 ViewModel，它可以取出 Model 的数据同时帮忙处理 View 中由于需要展示内容而涉及的业务逻辑。ViewModel中有一个专门的Binder（即双向数据绑定）用来负责数据的同步。这样即可以保证View和Model保持一致，也可以减少ViewModel的代码量。

                                                                                                                            分工：model主要负责数据模型，view主要负责用户界面视图展示，viewModel负责连接Model和View，是Model和View之间的通信桥梁。

                                                                                                                            优点：减少Controller大小，易维护，利用 MVVM 可以使我们的代码更专注于处理业务逻辑而不是去关心 DOM 操作。

                                                                                                                            缺点：各种 MVVM 组件和数据绑定之间的通信可能会让人筋疲力尽，在嵌套视图和复杂 UI 中操作视图模型及其状态是很困难的，数据绑定使得 Bug 很难被调试。

                                                                                                                            5、Data Property

                                                                                                                            组件的 data 选项是一个函数，Vue 会在创建新组件实例的过程中调用此函数。它应该返回一个对象，然后 Vue 会通过响应性系统将其包裹起来，并以 $data 的形式存储在组件实例中：

                                                                                                                            xxxxxxxxxx
                                                                                                                            const app = Vue.createApp({
                                                                                                                              data() {
                                                                                                                                return { count: 4 }
                                                                                                                              }
                                                                                                                            })
                                                                                                                            

                                                                                                                            const vm = app.mount('#app')
                                                                                                                            

                                                                                                                            console.log(vm.$data.count) // => 4
                                                                                                                            console.log(vm.count)       // => 4
                                                                                                                            

                                                                                                                            // 修改 vm.count 的值也会更新 $data.count
                                                                                                                            vm.count = 5
                                                                                                                            console.log(vm.$data.count) // => 5
                                                                                                                            

                                                                                                                            // 反之亦然
                                                                                                                            vm.$data.count = 6
                                                                                                                            console.log(vm.count) // => 6

                                                                                                                            注意：

                                                                                                                            1. 这些实例 property 仅在实例首次创建时被添加，所以你需要确保它们都在 data 函数返回的对象中。记住：只有当 Vue 实例被创建时所添加的 data 中返回的对象属性才是响应式的。
                                                                                                                            2. 将不包含在 data 中的新 property 添加到组件实例也是可行的。但由于该 property 不在背后的响应式 $data 对象内，所以 Vue 的响应性系统 不会自动跟踪它。
                                                                                                                            3. Vue 使用 $ 前缀通过组件实例暴露自己的内置 API。它还为内部 property 保留 _ 前缀。你应该避免使用这两个字符开头的顶级 data property 名称。

                                                                                                                            1、生命周期钩子

                                                                                                                            每个 Vue 实例在被创建时都要经过一系列的初始化过程——例如，需要设置数据监听、编译模板、将实例挂载到 DOM 并在数据变化时更新 DOM 等。在这个过程中会运行一些叫做 生命周期钩子 的函数，这给开发者在不同阶段添加自己的代码的机会。

                                                                                                                            针对不同的阶段主要有以下几个生命周期钩子函数：

                                                                                                                            创建阶段

                                                                                                                            • beforeCreate（创建前）
                                                                                                                            • created（创建后）
                                                                                                                            • beforeMount(挂载前)
                                                                                                                            • mounted（挂载后）

                                                                                                                            更新阶段

                                                                                                                            • beforeUpdate（更新前）
                                                                                                                            • updated（更新后）

                                                                                                                            卸载阶段

                                                                                                                            • beforeUnmount（vue2.0是beforeDestroy）（销毁前）
                                                                                                                            • unmounted（vue2.0是destroyed）（销毁后）

                                                                                                                            v-if

                                                                                                                            • v-if 是“真正”的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被 销毁和重建。

                                                                                                                            v-show

                                                                                                                            • 相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且是简单地基于 CSS 进行切换。

                                                                                                                            一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。 对应使用场景：

                                                                                                                            1. 如果需要非常 频繁地切换 ，则使用 v-show 较好；
                                                                                                                            2. 如果在运行时条件很少改变，则使用 v-if 较好。

                                                                                                                            不推荐同时使用 v-if 和 v-for，当 v-if 与 v-for 一起使用时，v-if 具有比 v-for 更高的优先级

                                                                                                                            状态维护

                                                                                                                            当 Vue 正在更新使用 v-for 渲染的元素列表时，它默认使用 “就地更新” 的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素，并且确保它们在每个索引位置正确渲染。

                                                                                                                            之前大家已经学过了相应的模版语法，我们可以在模版中使用表达式，虽然比较便利，不过还是建议模版内的表达式主要用来完成简单的运算，因为如果我们在模板中放入太多的逻辑，那么就会让模板过重且难以维护，例如：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <!DOCTYPE html>
                                                                                                                            <html lang="en">
                                                                                                                            

                                                                                                                            <head>
                                                                                                                              <meta charset="UTF-8">
                                                                                                                              <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                                                                                                              <title>Document</title>
                                                                                                                              <script src="https://unpkg.com/vue@next"></script>
                                                                                                                            </head>
                                                                                                                            

                                                                                                                            <body>
                                                                                                                              <div id="app">
                                                                                                                                {{ msg }} 转大写 {{ msg.toUpperCase().split('').join('-') }}
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                const app = Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      msg: "hellovue"
                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount('#app')
                                                                                                                              </script>
                                                                                                                            </body>
                                                                                                                            

                                                                                                                            </html>

                                                                                                                            以上代码中，我们对 msg 变量进行了小写转大写，然后又通过 split 函数对 msg 进行拆分，最后用 - 来连接。已经不是简单的一个表达式了，如果我们的逻辑更复杂一点的话，那么以上的表达式就不容易读懂了，而且因为字符串的处理是在以上DOM内的，如果我们需要在模版中多次引用的话也比较麻烦，因此对于逻辑复杂的结构，我们就不能用这种方式了，我们应该使用计算属性来完成。

                                                                                                                            关键字: computed

                                                                                                                            4.1、computed

                                                                                                                            在 Vue 对象内，我们可以通过添加 computed 属性来定义计算属性：

                                                                                                                                    }

                                                                                                                                  computed: {
                                                                                                                                    // 计算属性的 getter
                                                                                                                                    upperCaseMsg() {
                                                                                                                                      return this.msg.toUpperCase().split('').join('-')
                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount('#app')

                                                                                                                            4.2、计算属性缓存与方法

                                                                                                                            当然，在以上的例子中，我们也可以定义一个用来实现此功能的方法 （methods），然后通过方法来完成以上的转化：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                原始字符串：{{ msg }} <br>
                                                                                                                                转大写重新拼接： {{ upperCaseMsg() }}
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                const app = Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      msg: "hellovue"
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  methods: {
                                                                                                                                    upperCaseMsg() {
                                                                                                                                      return this.msg.toUpperCase().split('').join('-')
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  // computed: {
                                                                                                                                  //   // 计算属性的 getter
                                                                                                                                  //   upperCaseMsg() {
                                                                                                                                  //     return this.msg.toUpperCase().split('').join('-')
                                                                                                                                  //   }
                                                                                                                                  // }
                                                                                                                                }).mount('#app')
                                                                                                                              </script>

                                                                                                                            虽然可以通过方法来实现相同的效果，但是两种方式之间还是有差异的：

                                                                                                                            计算属性是基于它们的响应式依赖关系进行缓存的。只在相关响应式依赖发生改变时它们才会重新求值。这就意味着 只要 msg 没有发生改变，多次访问 upperCaseMsg 计算属性都会立即返回之前的计算结果，就不会再次执行函数。

                                                                                                                            我们为什么需要缓存？假设我们有一个性能开销比较大的计算属性 A，它需要遍历一个巨大的数组并做大量的计算。但是同时还有其他的计算属性依赖于 A。如果没有缓存，我们将不可避免的多次执行 A 的 getter 这样就会产生巨大的性能开销。

                                                                                                                            4.3、计算属性的setter

                                                                                                                            在以上的案例中，我们是给msg添加了计算属性，通过 getter 函数来返回需要的内容。默认情况下，计算属性只有 getter 函数，如果需要，我们也可以提供一个 setter 函数，不过在写法上与上面的写法有一定的区别：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                你的全名是：<input type="text" v-model="fullName"> <br>
                                                                                                                                姓氏：<input type="text" v-model="firstName"> <br>
                                                                                                                                名字: <input type="text" v-model="lastName"> <br>
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                const app = Vue.createApp({
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      firstName: 'Brad',
                                                                                                                                      lastName: 'Pitt'
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  computed: {
                                                                                                                                    fullName: {
                                                                                                                                      // getter
                                                                                                                                      get() {
                                                                                                                                        console.log('getter被调用')
                                                                                                                                        return this.firstName + ' ' + this.lastName
                                                                                                                                      },
                                                                                                                                      // setter
                                                                                                                                      set(newValue) {
                                                                                                                                        console.log('setter被调用')
                                                                                                                                        const name = newValue.split(' ')
                                                                                                                                        this.firstName = name[0]
                                                                                                                                        this.lastName = name[1]
                                                                                                                                      }
                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount('#app')
                                                                                                                              </script>

                                                                                                                            6、Class 与 Style 绑定

                                                                                                                            在页面设计过程中，我们一般都会操作元素的 class 列表和内联样式进行相应的绑定。 class 与 style 本身就是 HTML 元素的属性，因此就可以用 v-bind 来设置样式属性。

                                                                                                                            v-bind 用于 class 和 style 时，Vue.js 做了专门的增强， 除了字符串之外，还可以是 对象或数组，只需要通过表达式计算出字符串结果即可。

                                                                                                                            1. 对象绑定

                                                                                                                              • 数组 元素 即为 class 的值

                                                                                                                            7.2、事件修饰符

                                                                                                                            在事件处理程序中我们一般会调用 event.preventDefault() 或 event.stopPropagation() 来处理 DOM 事件细节，虽然我们可以通过这种方式去实现，不过 Vue 建议：方法只放纯粹的数据逻辑，而不是去处理 DOM 事件细节。因此，Vue.js 专门为 v-on 提供了事件修饰符来处理 DOM 事件细节。

                                                                                                                            之前提过，修饰符是由 . 开头的指令后缀来表示的：

                                                                                                                            • .stop -----> 将停止事件的传播，阻止它被分派到其他 Document 节点，等同于 JavaScript 中的 event.stopPropagation( )
                                                                                                                            • .prevent -----> 阻止元素发生默认的行为。等同于 JavaScript 中的 event.preventDefault( )，如果事件可取消，则取消该事件，但不停止事件的进一步传播。
                                                                                                                            • .capture -----> 使用捕获事件模式（从最不具体的元素到最具体的元素，即：根 ---> 叶子）
                                                                                                                            • .self -----> 只会触发自己范围内的事件，不包含子元素
                                                                                                                            • .once -----> 只会触发一次
                                                                                                                            • .passive -----> 不拦截默认事件，会告诉浏览器我们不想阻止事件的默认行为

                                                                                                                            如下案例，默认会按照冒泡型事件传播（ 叶子----> 根 ），那么可以通过 .stop 来停止事件的传播：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <!DOCTYPE html>
                                                                                                                            <html lang="en">
                                                                                                                            

                                                                                                                            <head>
                                                                                                                              <meta charset="UTF-8">
                                                                                                                              <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                                                                                                              <title>事件监听</title>
                                                                                                                              <script src="https://unpkg.com/vue@next"></script>
                                                                                                                            </head>
                                                                                                                            

                                                                                                                            <body>
                                                                                                                              <div id="app" style="text-align: center;">

                                                                                                                                  祖父元素
                                                                                                                                  <div @click="show('父元素')" style="background: hotpink; width: 50%;">
                                                                                                                                    父元素
                                                                                                                                    <div @click.stop="show('子元素')" style="background: yellowgreen; width: 50%;">
                                                                                                                                      子元素
                                                                                                                                    </div>
                                                                                                                                  </div>
                                                                                                                                </div>
                                                                                                                              </div>
                                                                                                                              <script>

                                                                                                                                  data() {
                                                                                                                                    return {}
                                                                                                                                  },

                                                                                                                                    show(msg) {
                                                                                                                                      console.log(msg);
                                                                                                                                    }
                                                                                                                                  }
                                                                                                                                }).mount("#app")
                                                                                                                              </script>
                                                                                                                            </body>

                                                                                                                            8、表单输入绑定

                                                                                                                            在之前的课程中，我们是通过 v-model 来给 <input> 表单控件进行数据的双向绑定，除了 <input> 以外，我们还可以给 <textarea> 及 <select> 、<checkbox>、<radio> 等元素上进行双向数据绑定，它会根据控件类型自动选取正确的方法来更新元素的。

                                                                                                                            注意 v-model 会忽略所有表单元素的 value、checked、selected 这些属性的初始值，总是将 Vue 实例的数据作为数据来源。对于数据来源，我们一般是在 JavaScript 在组件的 data 选项中声明初始值。

                                                                                                                            8.1、v-model

                                                                                                                            v-model 在内部为不同的输入元素使用不同的属性，并抛出不同的事件：

                                                                                                                                const app = Vue.createApp({

                                                                                                                                    }

                                                                                                                                  methods: {
                                                                                                                                    inputChanged() {
                                                                                                                                      console.log("您输入了：" + this.name);
                                                                                                                                    },

                                                                                                                                      console.log("正在输入中...");
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                }).mount("#app")
                                                                                                                              </script>

                                                                                                                            9、组件基础

                                                                                                                            9.1、认识组件

                                                                                                                            组件（Component）是 Vue.js 最强大的功能之一，通过组件可以帮我们扩展 HTML 元素，封装可重用的代码。

                                                                                                                            组件系统让我们可以用独立可复用的小组件来构建大型应用，几乎任意类型的应用的界面都可以抽象为一个组件树：

                                                                                                                            例如，我们的页面可能会有页头、侧边栏、内容区等组件，每个组件又包含了其它的像导航链接、博客之类的组件等。

                                                                                                                            为了能在模板中使用，这些组件必须先注册以便 Vue 能够识别。这里有两种组件的注册类型：全局注册和局部注册。

                                                                                                                            案例一：通过 component 方法全局注册的

                                                                                                                            一个简单的按钮计数的组件：

                                                                                                                            xxxxxxxxxx
                                                                                                                            <!DOCTYPE html>
                                                                                                                            <html lang="en">
                                                                                                                            

                                                                                                                            <head>
                                                                                                                              <meta charset="UTF-8">
                                                                                                                              <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                                                                                                              <title>组件</title>
                                                                                                                              <script src="https://unpkg.com/vue@next"></script>
                                                                                                                            </head>
                                                                                                                            

                                                                                                                            <body>
                                                                                                                              <div id="app">
                                                                                                                                <!-- 2、使用组件 -->
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                // 创建一个Vue 应用
                                                                                                                                const app = Vue.createApp({})
                                                                                                                                // 1、定义一个名为 counter-btn 的新全局组件
                                                                                                                                app.component('counter-btn', {
                                                                                                                                  data() {
                                                                                                                                    return {
                                                                                                                                      count: 0
                                                                                                                                    }
                                                                                                                                  },
                                                                                                                                  template: `
                                                                                                                                  <button @click="count++">你点击了 {{ count }} 次</button>
                                                                                                                                  `
                                                                                                                                })
                                                                                                                                app.mount("#app")
                                                                                                                              </script>
                                                                                                                            </body>
                                                                                                                            

                                                                                                                            </html>

                                                                                                                            组件名命名规范

                                                                                                                            用一个组件的时候，我们强烈推荐遵循 W3C 规范 来给自定义标签命名：

                                                                                                                            1. 全部小写
                                                                                                                            2. 包含连字符 (及：即有多个单词与连字符符号连接)

                                                                                                                            这样会帮助我们避免与当前以及未来的 HTML 元素发生冲突。

                                                                                                                            组件：是可复用的 Vue 实例，且带有一个名字，可以当作自定义元素来使用，在以上的例子中就是 <counter-btn></counter-btn>。

                                                                                                                            我们可以任意次数的复用组件：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                              </div>

                                                                                                                            注意：

                                                                                                                            • 当点击按钮时，每个组件都会各自独立维护它的 count。因为我们每用一次组件，就会有一个它的 新实例被创建。
                                                                                                                            • 组件是可复用的 Vue 实例，因此它们与 Vue.createApp( ) 时接收相同的选项，例如 data、computed、watch、methods 以及生命周期钩子等

                                                                                                                            9.2、组件的注册

                                                                                                                            每个组件在使用之前必须要先注册，这样 Vue 才能够识别的到。有两种方式来注册组件：

                                                                                                                            1. 全局注册

                                                                                                                              通过 app.component 来注册，它们在注册之后可以用在任何新创建的组件实例的模板中

                                                                                                                              xxxxxxxxxx
                                                                                                                              const app = Vue.createApp({...})
                                                                                                                              app.component('my-component-name', {
                                                                                                                                // ... 选项 ...
                                                                                                                              }

                                                                                                                            在以上的案例中，我们采用的就是全局注册的方式定义了组件 counter-btn，全局注册的组件可以用在其被注册之后的任何 (通过 Vue.createApp({})) 新创建的 Vue 根实例，也包括其组件树中的所有子组件的模板中。

                                                                                                                            1. 局部注册（var ComponentA = { ... }）

                                                                                                                              (1). 先通过一个普通的 JavaScript 对象来定义组件：

                                                                                                                              xxxxxxxxxx
                                                                                                                              const ComponentA = {
                                                                                                                                /* ... */
                                                                                                                              }

                                                                                                                              (2). 然后在 components 选项中定义你想要使用的组件：

                                                                                                                              xxxxxxxxxx
                                                                                                                              const app = Vue.createApp({
                                                                                                                                components: {
                                                                                                                                  'component-a': ComponentA
                                                                                                                                }
                                                                                                                              }).mount("#app")

                                                                                                                            如下案例，我们以局部注册的方式去改写以上计数器的案例：

                                                                                                                            xxxxxxxxxx
                                                                                                                              <div id="app">
                                                                                                                                <!-- 3、使用组件 -->
                                                                                                                                <counter-btn></counter-btn>
                                                                                                                              </div>
                                                                                                                              <script>
                                                                                                                                // 1、定义一个组件对象
                                                                                                                                const CounterBtn = {

                                                                                                                            9.3.3、单向数据流

                                                                                                                            所有的 prop 都使得其父子 prop 之间形成了一个 单向下行绑定：父级 prop 的更新会向下流动到子组件中，但是反过来则不行。这样会防止从子组件意外变更父级组件的状态，从而导致你的应用的数据流向难以理解。

                                                                                                                            另外，每次父级组件发生变更时，子组件中所有的 prop 都将会刷新为最新的值。这意味着你不应该在一个子组件内部改变 prop。如果你这样做了，Vue 会在浏览器的控制台中发出警告。

                                                                                                                            这里有两种常见的试图变更一个 prop 的情形：

                                                                                                                            1. 这个 prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用。在这种情况下，最好定义一个本地的 data property 并将这个 prop 作为其初始值：
                                                                                                                            xxxxxxxxxx
                                                                                                                            props: ['initialCounter'],
                                                                                                                            data() {
                                                                                                                              return {
                                                                                                                                counter: this.initialCounter
                                                                                                                              }
                                                                                                                            }
                                                                                                                            1. 这个 prop 以一种原始的值传入且需要进行转换。在这种情况下，最好使用这个 prop 的值来定义一个计算属性：
                                                                                                                            xxxxxxxxxx
                                                                                                                            props: ['size'],
                                                                                                                            computed: {
                                                                                                                              normalizedSize() {
                                                                                                                                return this.size.trim().toLowerCase()
                                                                                                                              }
                                                                                                                            }

                                                                                                                            9.4、监听子组件事件

                                                                                                                            父组件通过 props 可以传递数据给子组件，可以通过父级 prop 的更新会向下流动到子组件中，但是反过来则不行，但有时我们也需要与父级组件进行沟通，这是就要去监听子组件事件了。

                                                                                                                            如下，我们在之前的博客组件中，给博客列表中的每一项添加一个删除按钮，然后，当用户点击删除按钮时，会触发父组件的监听事件，父组件根据id来实现删除操作。因此，子组件所做的只是触发一个父组件关心的内部事件即可。

                                                                                                                            9.4.1、实现流程

                                                                                                                            实现流程如下，即：

                                                                                                                            1. 子组件使用 $emit(eventName) 触发事件
                                                                                                                            2. 父组件使用$on(eventName) 监听子组件通过 $emit 触发的事件

                                                                                                                            也就是说，子组件可以通过调用内建的 $emit 方法，以传入事件名称的方式来触发一个事件，而父组件可以在子组件上直接用 v-on 来监听子组件触发过来的事件。

                                                                                                                            首先我们现在 组件中添加一个删除按钮，并且给改按钮绑定单击事件，在单击事件中，子组件可以通过调用内建的 $emit 方法 并传入事件名称来触发一个事件：如下：

                                                                                                                            xxxxxxxxxx
                                                                                                                                app.component('blog-post', {
                                                                                                                                  // props 数组中的 title、blist用来接收父组件传递的值
                                                                                                                                  props: ['bid', 'btitle', ],
                                                                                                                                  template: `
                                                                                                                                    <p style="font-weight: 500">
                                                                                                                                      <span>{{bid}}. {{btitle}}</span>&nbsp;&nbsp;
                                                                                                                                      <!-- emit方法可以触发当前实例上的事件。第一个参数可接收事件名，第二个参数可传递的值-->
                                                                                                                                      <a href="#" style="color: red" @click="$emit('delete')">删除</a>
                                                                                                                                    </p>
                                                                                                                                  `
                                                                                                                                })

                                                                                                                            如上，当子组件上的删除按钮被点击时，就会触发名为 “delete” 的事件，那么后续我们只要在父组件给这个事件绑定对应方法，即可监听到子组件触发的事件了，完整代码如下：
