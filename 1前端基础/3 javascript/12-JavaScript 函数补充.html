JavaScript 函数补充

函数参数

形式参数与实际参数

通过我们之前的学习，大家应该对函数的参数已经比较熟悉了，来看以下例子：

function getSum(a, b) {
    return a + b;
}
getSum(10, 20); // 30

在函数定义语句中，定义在参数列表中的 a 和 b 便是 getSum 函数的形式参数，形式参数可以接收外部传入的实际参数的内容，以供函数内部访问使用。而当我们调用时传入的则是实际参数，如 getSum（10， 20）中的 10 和 20 就是实际参数。在调用过程中，会按照参数对应的顺序，把实际参数 10 赋值给形式参数 a；把实际参数 20 的值赋值给形式参数 b。

参数规则

  • （1） JavaScript 函数对参数的值不会进行任何的检查，并且不像其他语言中函数参数，它不会关注有多少个参数被传递，也不关注传递的参数的数据类型。

  • （2） 在JavaScript中，函数参数的默认值是 undefined。不过，在某些情况下设置不同的默认值还是有必要的。如下，还是以上的例子，如果调用函数时只传1个参数，那么最后的结果是多少：

function getSum(a, b) {
    return a + b;
}
getSum(10);

如果我们在调用 getSum 时并未传第2个参数，那么此时在进行运算后，就会返回 NaN。主要是由于运算过程中，因为没有给形式参数 b 传值，因此 b 的值就默认为 undefined 了，所以是 10 + undefined 运算的结果就是 NaN 了。

针对以上的情况，去我们就有必要给两个参数设置默认值。当调用时如果没有传值（即值为 undefined 时），那么需要给他们设置默认值 0：

function getSum(a, b) {
    if (a == undefined) {
        a = 0
    }
    if (b == undefined) {
        b = 0
    }
    return a + b;
}
getSum(10) // 10

当然，在es6中，我们可以给函数的参数设置默认值，以上代码也可以简化成：

function getSum(a = 0,b = 0) {
    return a + b;
}
alert(getSum(10)) // 10

arguments对象

在JavaScript的函数中，还有一个内置的 arguments 对象，函数的实际参数会被保存在一个类似数组的arguments对象中。在函数内，我们可以通过像数组元素的访问方式来找出传入的参数：arguments[i]，第一个传来的参数是 arguments[0]，以此类推。对于参数的数量，我们可以通过arguments.length 来获取。

function getSum(a, b) {
    return arguments[0] + arguments[1];
}
alert(getSum(10, 20)) // 30

函数作用域

在 JavaScript中，函数内定义的变量是不能在函数之外的任何地方访问的，因为这些变量仅仅在该函数的域的内部（局部作用域）有效。而一个函数是可以访问定义在其范围内的任何变量和函数；定义在全局域中的函数可以访问所有定义在全局域（全局作用域）中的变量。

var num1 = 10, num2 = 20; // num1 h和 num2 在全局作用域定义的,即为全局变量 
function mutiply() {
     return num1 * num2; // 函数内部可以访问全局变量 num1 和 num2
} 
console.log(mutiply()); // 输出结果为: 200

以上代码中，我们可以在函数中访问全局变量。而我们无法在全局作用域中去访问局部变量，一旦这样做了就会报错：

var num1 = 10, num2 = 20; // num1 h和 num2 在全局作用域定义的,即为全局变量
function mutiply() {
    var result = num1 * num2; // 将num1 和 num2 的运算结果赋值给局部变量 result
    return result;
}
mutiply();
console.log(result); // 访问局部变量 result

以上代码中，因为 result 是局部变量，因此报错。

注意，一般情况下，我们都需要使用 var 来声明一个变量，因此如果我们在函数内部没有用var关键字来声明变量，那么该变量就会被提升为全局变量：

var num1 = 10, num2 = 20; // num1 h和 num2 在全局作用域定义的,即为全局变量
function mutiply() {
    result = num1 * num2; // 将num1 和 num2 的运算结果赋值给局部变量 result
    return result;
}
mutiply();
console.log(result); // 访问局部变量 result

以上代码中，result是全局变量，因此，控制台输出：200。

全局作用域

在代码中任何地方都能访问到的对象拥有全局作用域，一般来说以下几种情形拥有全局作用域：

  • 最外层函数或者是在最外层函数外面定义的变量拥有全局作用域；
  • 所有未通过var关键字定义而直接赋值的变量将自动被声明为全局变量，如以上案例中的 result；
  • 所有window对象的属性拥有全局作用域。

总之，作用域是分层的，内层作用域可以访问外层作用域的变量，而外层作用域则无法直接访问内层作用域中的变量。如下，我们用不同颜色的气泡来比喻作用域，那么：

作用域链

我们已经知道了在通常情况下，外层是无法直接访问函数内定义的局部变量的，但是并不是说我们完全不能获取函数的局部变量，我们可以在函数内部再定义一个函数：

var num1 = 10,
    num2 = 20; // num1 h和 num2 在全局作用域定义的,即为全局变量
function mutiply() {
    var result = num1 * num2; // 将num1 和 num2 的运算结果赋值给局部变量 result
    function getResult() {
        return result;
    }
}

在上面的代码中，函数 getResult 就被包裹在函数 mutiply 内部，这时 mutiply 内部的所有局部变量，对 getResult 都是可见的。不过如果反过来就不成立了，getResult 内部的局部变量对 mutiply 就是不可见的。这就是 Javascript 语言特有的 "链式作用域" 结构（chain scope），子对象会一级一级地向上寻找所有父对象的变量，因此，父对象的所有变量，对子对象都是可见的，反之则不成立。

既然 getResult 函数可以读取 mutiply 中的局部变量，那么我们只要把局部变量封装在 getResult 内部，然后再把 getResult 当做 mutiply 的返回值返回，这样外部不就可以读取 multiply 的局部变量了嘛！

var num1 = 10, num2 = 20;
function mutiply() {
    var result = num1 * num2;
    function getResult () {
        return result;
    }
    return getResult; // 将函数 getResult 作为返回值返回
}
var getRes = mutiply(); // 将返回的函数赋值给 getRes 变量,即让 getRes指向原来的 getResult函数
console.log("result: " + getRes()); // 运行 getRes 即可获取 getResult 所返回的局部变量了

闭包

闭包（closure）是JavaScript语言的一个难点，但同时也是它的特色，很多高级应用都要依靠闭包实现。

一个函数和对其周围状态（lexical environment，词法环境）的引用捆绑在一起（或者说函数被引用包围），这样的组合就是闭包（closure）。

通过闭包，可以让我们在一个内层函数中访问其外层函数的作用域。因此，每当我们创建一个函数时，闭包就会在函数创建的同时被创建出来。

我们之前的案例中：

var num1 = 10, num2 = 20;
function mutiply() {
    var result = num1 * num2;
    function getResult () {
        return result;
    }
    return getResult; // 将函数 getResult 作为返回值返回
}
var getRes = mutiply(); // 将返回的函数赋值给 getRes 变量,即让 getRes指向原来的 getResult函数
console.log("result: " + getRes()); // 运行 getRes 即可获取 getResult 所返回的局部变量了

我们使用 getResult 函数实现了将原本外界无法访问的 result 的值给返回，getResult 就形成了闭包，它拥有对 mutiply() 函数作用域的引用，可以理解为就是定义在一个函数内部的函数，不过这个函数并不会立即执行，需要等到被调用了才执行。因此，闭包本质上就是将函数内部和函数外部连接起来的一座桥梁。

闭包使用场景

闭包可以用在许多场景中，常见的两个场景如下：

  • （1） 读取函数内部的变量；
  • （2） 让局部变量的值始终保持在内存中。（注意：闭包只会保存被子作用域引用的变量，对于未被使用到的变量仍会被回收）
var num1 = 10, num2 = 20;
function mutiply() {
    var result = num1 * num2;
    function getResult () {
        return result;
    }
    return getResult; // 将函数 getResult 作为返回值返回
}
var getRes = mutiply(); // 将返回的函数赋值给 getRes 变量,即让 getRes指向原来的 getResult函数
console.log("result: " + getRes()); // result: 200
console.log("result: " + getRes()); // result: 200

我们来看第二种场景。在一些编程语言中，一个函数中的局部变量仅存在于此函数的执行期间。因此，当 mutiply () 执行完毕，很多人可能就会认为局部变量 result 就不能再被访问。

而在以上代码中，在第一句 log 之前我们已经执行了 mutiply 函数，而当我们再执行接下去的两句 log 时仍然可以正常访问到 result 的值。JavaScript中的函数会形成闭包，闭包是由函数以及声明该函数的词法环境组合而成的，且该环境包含了这个闭包创建时作用域内的任何局部变量。原因就在于 mutiply 是 getResult 的父函数，并且最终 getResult 被赋给了全局变量 getRes，这导致 getResult 的实例维持了一个对它的词法环境（变量 name 存在于其中）的引用，因此当 getRes 被调用时，变量 result被引用了（return result），因此不会被垃圾回收机制（garbage collection）回收，所以仍然可以使用。

再来深入了解闭包的用法：

function add(a) {
    return function (b) {
        return a + b
    }
}
var num1 = add(2)
var num2 = add(10)
console.log(num1(3)); // 5
console.log(num2(30)); // 40

注意事项

  • 闭包会使得函数中的变量都被保存在内存中，这样内存开销就上去了，因此不能滥用闭包，否则会造成网页的性能问题。要想避免此问题，我们可以在退出函数之前，将不使用的局部变量全部删除。
  • 闭包会在父函数外部改变父函数内部变量的值。

对象

还记得我们之前给大家介绍了 JavaScript 中的数据类型吧？基本类型和引用类型。在 JavaScript 中，当我们将一个对象赋值给某个变量时，JavaScript 引擎会在内存中生成一个对象，然后将对象的内存地址赋值给变量，变量实际上保存的是对象的引用。

如下:

var obj = { foo: 5}

如上，obj 保存的是一个地址（reference），因此当我们需要访问其中的属性 foo 时，要经过以下几个步骤：

  1. JavaScript 引擎首先需要从 obj 拿到内存地址；
  2. 然后再根据该地址去获取出原始对象；
  3. 最后返回它的 foo 属性。

而 JavaScript 中的原始对象，一般是以字典（键值对）的结构来保存的，每一个属性名都对应一个属性表述对象，例如 foo 属性，实际上是按照如下形式进行保存的：

￼

即： 

{
  foo: {
    [[value]]: 5
    [[writable]]: true
    [[enumerable]]: true
    [[configurable]]: true
  }
}

一般属性的值是保存在该属性表述对象的 value 属性中的。

函数

接着我们再来看一下，对象中的方法：

var obj = { foo: function () {} };

如果对象中的属性所对应的值是一个方法，那么此时，JavaScript 引擎会先将函数单独保存在内存中，然后再将函数的地址值赋值给 foo 属性的 value 属性, 如下：

{
  foo: {
    [[value]]: 函数的地址
    [[writable]]: true
    [[enumerable]]: true
    [[configurable]]: true
  }
}

￼

那么，因为函数是一个单独的对象，所以它可以在不同的环境（执行上下文）中去执行，如下。

var f = function () {};
var obj = { f: f };

// 单独执行
f()

// obj 环境执行
obj.f()

环境变量

JavaScript 允许在函数体内部引用当前环境的其他变量，那么又因为函数是可以在不同的运行环境执行，所以就需要有一种机制，能够在函数体内部获得当前的运行环境（context即上下文），然后才可以引用到对应环境中的变量。如何获取呢？ 此时就需要使用 this 了，它专门用于函数体内部，用来指代函数当前的运行环境的。

var f = function () {
  console.log(this.x); // 输出当前运行环境中的x
};
